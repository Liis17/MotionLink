# Протокол MotionLink (UDP)

Parent: [[index]]

> Полная спецификация — `PROTOCOL.md` в корне репозитория. Этот файл — краткая выжимка для навигации.

## Назначение
Клиент находит сервер в LAN и шлёт ему высокочастотный поток данных датчиков. **Только UDP.** Никаких TCP / WebSocket / HTTP.

## Этап 1. Discovery

| Параметр | Значение |
|----------|----------|
| Транспорт | UDP-broadcast |
| Адрес | `255.255.255.255:58930` |
| Source-порт | произвольный эфемерный (`bind 0.0.0.0:0`) |
| Сокет-опции | `SO_BROADCAST=1`, `SO_REUSEADDR=1` |
| Период | 1 раз в секунду, бесконечно |
| Payload пробы | `MOTIONLINK_DISCOVER\n` (20 байт ASCII) |
| Ответ сервера | JSON `{"name": "...", "port": 58930}` — оба поля опциональные |

IP сервера берётся из `sockaddr` ответного пакета, **не** из JSON.

Реализация клиента: [[modules/ios-discovery]].

## Этап 2. Поток данных

| Параметр | Значение |
|----------|----------|
| Транспорт | UDP unicast на `host:port` из discovery |
| Частота | ~30 Гц (один пакет каждые ~33 мс) |
| Размер | ~200–300 байт |
| Формат | один JSON-объект на датаграмму, UTF-8, компактный |
| Подтверждения | нет; клиент не читает ответ |

### Структура пакета
См. [[api/motion-snapshot]].

```json
{
  "t": 1779993600.123456,
  "roll": -12.4, "pitch": 34.1, "yaw": 87.2,
  "acceleration": {"x": 0.012, "y": -0.143, "z": 0.981},
  "gravity":      {"x": 0.0,   "y": 0.0,    "z": -1.0},
  "rotationRate": {"x": 0.002, "y": -0.001, "z": 0.004}
}
```

### Единицы измерения
- `roll/pitch/yaw` — **градусы** (CoreMotion отдаёт радианы, клиент конвертирует через `180/π`).
- `acceleration`, `gravity` — **g** (1 g ≈ 9.81 м/с²).
- `rotationRate` — **рад/с**.

### Система координат
Портретная ориентация iPhone (экран к лицу):
- X — вправо, Y — вверх, Z — из экрана к лицу.
- `yaw` — **не компас**, а смещение от стартовой ориентации (`xArbitraryZVertical`).

## Поведение клиента при сбоях
- `sendto` ошибка → лог, продолжаем слать.
- HUD-индикатор зелёный = `NWConnection` в `.ready`. UDP без рукопожатия → это **не** означает, что сервер реально слушает.
