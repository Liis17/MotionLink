### Нюансы, которые легко забыть

- **`onDispose` в DisposableEffect** — захвачена локальная переменная `active = state`, иначе cleanup использует уже новое состояние. См. [[modules/android-app]] § Важная деталь.
- **MulticastLock** обязателен на многих устройствах для приёма broadcast-ответов. См. [[modules/android-discovery]].
- **Android yaw — компасный** (через ROTATION_VECTOR + магнитометр), в отличие от iOS `xArbitraryZVertical`. Поведение немного разойдётся. См. [[modules/android-motion]].
- **org.json `optString` не возвращает null** — приходится отдельно проверять `has` + `isNull`. См. [[modules/android-discovery]] § Парсинг ответа.

### Инцидент сборки

При первой сборке: `Cannot add extension with name 'kotlin', as there is an extension already registered with that name. Failed to apply plugin 'org.jetbrains.kotlin.android'.`

**Причина**: AGP 9.2.1 включает built-in Kotlin support — сам регистрирует `kotlin {}` extension через `KotlinAndroidExtension`. Применение `org.jetbrains.kotlin.android` параллельно пытается зарегистрировать тот же extension повторно → конфликт.

**Фикс**: убрать `kotlin.android` из плагинов (и из top-level `build.gradle.kts`, и из `app/build.gradle.kts`). Оставить только `kotlin.compose` — он не создаёт `kotlin` extension, а только добавляет compose compiler в Kotlin-компиляцию.

После фикса `BUILD SUCCESSFUL` (Gradle 9.4.1, AGP 9.2.1).
